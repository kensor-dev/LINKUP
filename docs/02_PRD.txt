LINKUP — Product Requirements (PRD)
=====================================

Пометки:
  [MVP]  — нужно для первого запуска
  [CORE] — добавляется сразу после MVP
  [PLUS] — следующий этап


=======================================================
БЛОК 1 — GPS-ТРЕКИНГ
=======================================================

1.1 ПРИЛОЖЕНИЕ КУРЬЕРА (мобильное)  [MVP]
------------------------------------------
Платформа: React Native + Expo (iOS + Android)

Экраны:
  Вход
    Ввод номера телефона → получить SMS-код → войти
    Без паролей: курьеры не хотят с ними возиться

  Мои заказы
    Список назначенных заказов на сегодня
    Карточка: адрес, имя клиента, статус
    Pull-to-refresh

  Детали заказа
    Полный адрес доставки
    Имя и телефон клиента
    Состав заказа
    Кнопка "Открыть в картах" (2GIS / Яндекс)
    Кнопки смены статуса:
      [Забрал товар] → [Еду] → [Доставил] / [Не застал]

GPS-трансляция:
  Кнопка "Начать смену" → координаты отправляются каждые 5 секунд
  Кнопка "Завершить смену" → трансляция останавливается
  Работает при свёрнутом приложении (background mode)
  Буферизация: нет интернета → координаты копятся → отправляются пакетом

Push-уведомление при назначении нового заказа.

Требования: Android 8+, iOS 14+. Работает на дешёвых Android-телефонах.


1.2 КАРТА С КУРЬЕРАМИ В РЕАЛЬНОМ ВРЕМЕНИ (дашборд)  [MVP]
-----------------------------------------------------------
Карта 2GIS:
  Маркеры всех активных курьеров — обновляются через WebSocket
  Клик на курьер → имя, текущий заказ, когда последний раз был онлайн

Боковая панель:
  Список всех курьеров: онлайн / офлайн, статус текущего заказа

Счётчики:
  Заказов сегодня / В доставке / Выполнено


1.3 ЖИВАЯ ССЫЛКА ДЛЯ КЛИЕНТА  [MVP]
--------------------------------------
При создании заказа генерируется уникальная ссылка: linkup.ru/track/abc123

Клиент открывает в браузере (без приложения, без регистрации):
  Карта с курьером — обновляется автоматически
  Имя курьера
  Статус заказа
  Примерное время прибытия (ETA)

Страница брендирована под бизнес: логотип + фирменный цвет.
Мобильная вёрстка в приоритете.
Ссылка активна пока заказ не доставлен.


1.4 АВТОМАТИЧЕСКИЕ УВЕДОМЛЕНИЯ КЛИЕНТУ  [CORE]
------------------------------------------------
Триггеры и сообщения:

  Курьер назначен:
    "Ваш заказ передан курьеру [Имя]. Отслеживайте: [ссылка]"

  Курьер выехал (статус "В пути"):
    "Курьер выехал, будет примерно через [N] минут"

  Курьер в 500м от адреса:
    "Курьер почти у вас! Приготовьтесь принять заказ"

  Заказ доставлен:
    "Заказ доставлен. Спасибо!"

  Курьер задерживается (ETA сдвинулся на 15+ минут):
    "Курьер немного задерживается. Новое время: [N]"

Каналы (настраиваются бизнесом):
  SMS — через SendPulse
  WhatsApp — через WhatsApp Business API


1.5 УПРАВЛЕНИЕ ЗАКАЗАМИ (диспетчерский модуль)  [MVP]
------------------------------------------------------
Создать заказ:
  Телефон клиента → автозаполнить имя если уже есть в базе
  Адрес доставки (с подсказками)
  Состав заказа и сумма
  Примечание
  Назначить курьера (или оставить без назначения)

Список заказов:
  Фильтры: статус, курьер, дата
  Поиск: номер заказа, телефон клиента
  Переназначить курьера

Статусы заказа (жизненный цикл):
  новый → назначен → забрал курьер → в пути → доставлен / не доставлен / отменён

Детали заказа:
  История статусов с временем каждого изменения
  Мини-карта с позицией курьера (live)
  Ссылка для клиента (скопировать одной кнопкой)


1.6 АНАЛИТИКА КУРЬЕРОВ  [CORE]
--------------------------------
За выбранный период (день / неделя / месяц):

  По каждому курьеру:
    Количество доставок
    Среднее время доставки (от "забрал" до "доставил")
    % доставок вовремя vs с опозданием
    Количество случаев "не застал клиента"

  Общий рейтинг курьеров (таблица, сортируется по эффективности)

  Пиковые часы нагрузки (помогает планировать смены)


1.7 УПРАВЛЕНИЕ СМЕНАМИ КУРЬЕРОВ  [CORE]
-----------------------------------------
  Расписание смен: какой курьер работает в какие дни и часы
  История смен: дата, начало/конец, количество заказов за смену
  Кто сейчас на смене (список на дашборде)
  Курьер не может получать заказы если не на смене


=======================================================
БЛОК 2 — CRM
=======================================================

2.1 АВТОМАТИЧЕСКИЙ ПРОФИЛЬ КЛИЕНТА  [MVP]
------------------------------------------
Создаётся автоматически при первом заказе с номером телефона.
Ничего вводить вручную не нужно.

Что хранится:
  Контакты: телефон, имя, email
  Список адресов доставки (из истории заказов)
  Сегмент (считается автоматически)
  Теги (добавляются вручную)
  Заметки (добавляются вручную)
  CRM-метрики: количество заказов, LTV, средний чек, даты


2.2 ИСТОРИЯ ЗАКАЗОВ КЛИЕНТА  [MVP]
------------------------------------
В карточке клиента — полный список всех заказов:
  Дата, сумма, состав, курьер, статус доставки
  Был ли курьер вовремя (связь с данными GPS)
  Клик на заказ → открывает детали


2.3 СЕГМЕНТЫ КЛИЕНТОВ  [MVP]
------------------------------
Пересчитываются автоматически каждую ночь:

  Новый      — 1–2 заказа
  Постоянный — 3+ заказа, последний заказ < 30 дней назад
  Засыпает   — последний заказ 30–90 дней назад
  Потерян    — последний заказ > 90 дней назад

Видны в списке клиентов и карточке клиента.
Фильтр по сегменту в списке клиентов.


2.4 ТЕГИ И ЗАМЕТКИ  [MVP]
---------------------------
  Теги: свободные метки на клиента (например: "аллергия", "vip", "утром")
  Заметки: произвольный текст от менеджера
  Оба поля — редактируются прямо в карточке клиента


2.5 АНАЛИТИКА CRM  [CORE]
---------------------------
  Динамика клиентской базы по сегментам (график)
  Retention rate: % клиентов, вернувшихся за 30 / 60 / 90 дней
  LTV в разбивке по периодам
  Среднее время между заказами одного клиента
  Эффективность сценариев: сколько сработало, сколько конвертировало


2.6 ВИЗУАЛЬНЫЙ КОНСТРУКТОР АВТОСЦЕНАРИЕВ  [CORE]
-------------------------------------------------
Интерфейс: ЕСЛИ [триггер] → ТО [действие].
Без кода, для менеджера.

Триггеры (GPS-связанные — уникальная фишка):
  Курьер опоздал на N минут
  Заказ получил статус "не застал"
  Доставка заняла в 2 раза дольше обычного

Триггеры (CRM):
  Клиент не заказывал N дней
  Клиент сделал первый заказ
  Клиент достиг N-го заказа
  LTV клиента пересёк порог
  День рождения клиента (если известен)

Действия:
  Отправить SMS (по шаблону)
  Отправить WhatsApp (по шаблону)
  Создать задачу для менеджера
  Добавить тег клиенту

Готовые шаблоны (стартовать не с пустого экрана):
  "Курьер опоздал → промокод"
    ЕСЛИ опоздание > 30 мин → WhatsApp клиенту с промокодом -10%
  "Реактивация засыпающего"
    ЕСЛИ нет заказа 30 дней → задача менеджеру + SMS со скидкой
  "Приветствие нового клиента"
    ЕСЛИ первый заказ доставлен → WhatsApp "Спасибо!"
  "VIP порог достигнут"
    ЕСЛИ LTV > 10 000 ₽ → задача менеджеру + тег "vip"

Логи: история срабатываний каждого сценария с деталями.


2.7 ЗАДАЧИ ДЛЯ МЕНЕДЖЕРОВ  [CORE]
------------------------------------
  Создаётся автоматически из сценария или вручную из карточки клиента
  Список задач: сегодня / просроченные / все
  Назначить на конкретного менеджера
  Отметить выполненной + комментарий
  История задач по клиенту видна в его карточке


=======================================================
БЛОК 3 — ОБЩИЕ СИСТЕМНЫЕ ФИЧИ
=======================================================

РЕГИСТРАЦИЯ И ВХОД  [MVP]
  Регистрация: название компании, email, пароль
  Верификация email (ссылка на почту)
  Вход: email + пароль
  Роли: владелец / диспетчер
  Пригласить сотрудника по email

НАСТРОЙКИ КОМПАНИИ  [MVP]
  Название, логотип, фирменный цвет (для страницы трекинга)
  Настройки уведомлений (какие SMS/WhatsApp отправлять клиентам)

УПРАВЛЕНИЕ КУРЬЕРАМИ  [MVP]
  Добавить курьера: имя, телефон, фото
  Деактивировать курьера
  Список всех курьеров
